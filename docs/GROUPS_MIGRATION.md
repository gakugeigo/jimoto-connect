# マイグループ機能のマイグレーション（詳細手順）

Phase 3.5 のマイグループ機能を使うには、以下のマイグレーションを実行してください。

---

## ステップ1: マイグレーションファイルを開く

1. **Cursor（またはエディタ）** でプロジェクトを開く
2. 左のファイル一覧で、次のファイルをクリックして開く：
   ```
   sns-app / supabase / migrations / 20260219120000_create_groups.sql
   ```
3. ファイルの中身が表示される

---

## ステップ2: SQL をすべてコピーする

1. 開いたファイル内で **Ctrl + A**（Mac は Cmd + A）を押す → 全文が選択される
2. **Ctrl + C**（Mac は Cmd + C）を押す → クリップボードにコピーされる

---

## ステップ3: Supabase ダッシュボードを開く

1. ブラウザで **https://supabase.com/dashboard** を開く
2. ログインする（まだの場合）
3. **Jimoto-Connect 用のプロジェクト** をクリックして選択する

---

## ステップ4: SQL Editor を開く

1. 左側のメニューを確認する
2. **「SQL Editor」** という項目をクリックする
   - アイコンは「</>」やコードのようなマーク
   - 「Table Editor」の下あたりにあることが多い
3. SQL Editor の画面が開く

---

## ステップ5: 新しいクエリを作成する

1. 画面上部に **「New query」** というボタンがある
2. **「New query」** をクリックする
3. 真っ白な入力エリアが表示される

---

## ステップ6: SQL を貼り付けて実行する

1. 入力エリア内をクリックしてカーソルを置く
2. **Ctrl + V**（Mac は Cmd + V）を押す → コピーした SQL が貼り付けられる
3. 画面右下の **「Run」** ボタンをクリックする
   - または **Ctrl + Enter**（Mac は Cmd + Enter）でも実行できる

---

## ステップ7: 結果を確認する

- **成功した場合**  
  画面下部に「Success. No rows returned」や緑色のメッセージが表示される

- **エラーが出た場合**  
  赤い文字でエラーメッセージが表示される  
  → その内容をコピーして、開発者に共有するか、エラー内容を確認する

---

## 完了後

マイグレーションが成功したら、アプリの **「マイグループ」** からグループを作成できるようになります。
