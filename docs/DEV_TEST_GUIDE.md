# 開発時の動作確認ガイド

登録者が少ない状態で検索・プロフィール機能を確認する方法です。

---

## 方法1: 開発用テストデータを投入（推奨）

1. **Supabase ダッシュボード** を開く
2. **SQL Editor** を開く
3. `supabase/seeds/dev_test_profiles.sql` の内容をコピーして実行

これで以下の3人のテストプロフィールが追加されます：

| 名前       | 居住地     | 学校             | 卒業年 | 部活   |
|------------|------------|------------------|--------|--------|
| 山田 花子  | 東京都     | 那覇高等学校     | 1992年 | 帰宅部 |
| 佐藤 太郎  | 東京都     | 那覇高等学校     | 1992年 | 野球部 |
| 鈴木 次郎  | 神奈川県横浜市 | 横浜翠嵐高等学校 | 1990年 | サッカー部 |

**確認手順：**
- 自分で「那覇高等学校」または「横浜翠嵐高等学校」を登録している場合、同窓生検索で上記がヒットします
- 学校を指定しないで「東京都」で検索 → 山田・佐藤がヒット
- 検索結果をクリック → 他ユーザーのプロフィールが表示される

## 方法1b: 奈良中・中大附属の同級生20人を投入（同級生マップ用）

**奈良中学校（横浜市青葉区）・中央大学附属高校（小金井市）** を登録している場合、同級生マップに表示されます。

1. **schools.sql を実行**（奈良中学校・中央大学附属高校が DB に無い場合のみ）
   - `supabase/seeds/schools.sql` の内容を SQL Editor で実行
   - ⚠️ 注意: 既存の schools は削除され、profile_schools も連鎖削除されます。実行後はオンボーディングで学校を再登録してください。

2. **デモシードを実行**
   - `supabase/seeds/supabase_demo_seed.sql` の内容を SQL Editor で実行（推奨・1ファイルで完結）
   - または `supabase/seeds/demo_10_profiles.sql` を実行

これで **合計20人** が追加されます（別々のプロフィール）：
- **10人**: 奈良中学校 1989年卒のみ（出身地は横浜市、居住地は東京・千葉・神奈川・埼玉に分散）
- **10人**: 中央大学附属高校 1992年卒のみ（出身地は東京・千葉・埼玉・神奈川、男子校の部活）

**確認手順：**
- 同級生マップで「奈良中学校」「1989年」を選択 → 10人が神奈川・東京・千葉・埼玉に表示
- 「中央大学附属高校」「1992年」を選択 → 別の10人が同様に表示

## 方法2: 複数アカウントで登録

1. 別のメールアドレスで Clerk から新規登録
2. オンボーディングで学校・部活を登録
3. 元のアカウントで同窓生検索 → 2人目がヒット

---

## 検索のヒット条件

- **学校**: 自分が登録している学校のみ検索条件に表示（同じ学校の人がヒット）
- **卒業年・部活・居住地**: 指定するとさらに絞り込み
