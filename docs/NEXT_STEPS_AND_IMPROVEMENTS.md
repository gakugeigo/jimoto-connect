## 2. 改善点 10個

### 機能・UX

1. **部活を選択式＋自由追加に** ✅ 完了
   - 現状: 自由入力のみ
   - 改善: よくある部活を選択肢にし、「その他」で自由入力（業種・職種と同様）

2. **同級生の居住地一覧** ✅ 完了
   - 現状: 検索結果で個別に表示
   - 改善: 都道府県ごとの人数や一覧を専用画面で表示（`/residents`）

3. **画像アップロード** ⏭️ スキップ（不要）
   - 現状: 投稿に画像URLはあるが、アップロードUIなし
   - 改善: Supabase Storage を使った画像アップロード・プレビュー

4. **通知・未読バッジ** ⏭️ スキップ（不要）
   - 現状: いいね・コメント・招待があっても気づきにくい
   - 改善: ヘッダーに通知アイコン、未読件数のバッジ表示

5. **検索結果のソート・ページネーション** ✅ 完了
   - 現状: 検索結果は表示順が固定
   - 改善: 帰省中優先、居住地順などソート機能、件数が多い場合はページネーション

### UI・デザイン

6. **ローディング・スケルトン表示** ✅ 完了
   - 現状: データ取得中に白画面や一瞬のチラつき
   - 改善: スケルトンUIやローディングスピナーで体感速度を向上

7. **エラー表示の統一** ✅ 完了
   - 現状: `alert()` でエラー表示
   - 改善: トーストやインラインエラーメッセージで統一、ユーザーフレンドリーに

8. **モバイルファーストの見直し**
   - 現状: PC を前提にしたレイアウト
   - 改善: スマホでのタップ領域、フォントサイズ、ナビゲーションの最適化

### 技術・品質

9. **API Routes の追加（要件対応）**
   - 現状: Server Actions のみ
   - 改善: 外部連携や Webhook 用に API Routes を追加し、要件の「API Routes 推奨」に合わせる

10. **テストの追加**
    - 現状: テストなし
    - 改善: 主要な Server Actions やコンポーネントに Jest / React Testing Library でテストを追加
