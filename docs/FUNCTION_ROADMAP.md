# Jimoto-Connect 機能ロードマップ

UI/UX は後回しにし、**基本機能を優先**して実装する方針です。

| 機能 | 状態 | 備考 |
|------|------|------|

---

## 実装優先順位（推奨）

### Phase 1: 投稿の完成（最優先）
1. **投稿一覧をDBから取得して表示** - ダッシュボードで実際の投稿を表示
2. **投稿作成後の一覧更新** - 投稿後に一覧が更新されるようにする

### Phase 2: いいね・コメント
3. **いいね** - 投稿へのいいね/いいね解除
4. **コメント** - 投稿へのコメント追加・表示

### Phase 3: 検索・つながり
5. **同窓生検索** - 学校・卒業年・部活・居住地で検索
6. **他ユーザープロフィール** - 検索結果からプロフィール閲覧

### Phase 3.5: マイグループ（帰宅部向け）
7. **マイグループ** - 部活外の任意グループ作成・招待・投稿（LINE風）
   - 帰宅部の人もクラス以外でつながれる
   - 招待リンク or ユーザー検索でメンバー追加

### Phase 4: イベント
8. **同窓会イベント** - イベント作成・参加申し込み・一覧表示
