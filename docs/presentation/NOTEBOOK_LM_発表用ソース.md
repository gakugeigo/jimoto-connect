## スライド1: タイトル

**Jimoto-Connect**
懐かしいあの場所に、同郷の仲間を。

同窓生・同郷の仲間を見つける SNS

---

## スライド2: 課題

**解決する課題**
- 都会での孤独感：地方から上京した人が、心理的に安心できる「同郷の仲間」が近くに見つからない
- 地元コミュニティの希薄化：地元の友人がどこに住んでいるかわからず、疎遠になる
- 既存SNSの限界：Facebook 等では「今、近くに住んでいる同郷の人」での検索が難しい

**課題が発生する状況**
- 上京直後の心細い時期
- 職場と家の往復だけの生活に疲れた時
- 地元のニュースや方言に触れて懐かしくなった時

---

## スライド3: 解決策

**Jimoto-Connect が提供する価値**

1. 「出身地」×「現在地」のクロス検索
   - 出身校・卒業年・部活・居住地で同窓生を探せる

2. 母校掲示板（デジタル居酒屋ノート）
   - 学校ごとの掲示板で気軽に書き込み・いいね・コメント

3. 帰省中チェックイン
   - 「今地元に帰省中」と伝えて、近くの同級生とつながる

4. 居住地マップ
   - 都道府県ごとに同級生の分布を可視化

## スライド9: 技術構成

**使用サービス**

| 用途 | サービス |
|------|----------|
| フロントエンド | Next.js 16, TypeScript, Tailwind CSS |
| 認証 | Clerk（サインアップ/ログイン） |
| データベース | Supabase（PostgreSQL） |
| ストレージ | Supabase Storage |
| デプロイ | Vercel |

**アーキテクチャ**
ユーザー → Next.js (Vercel) → Clerk（認証）
                    ↓
              Supabase（DB・Storage）

## スライド10: ビルド&デプロイで工夫した点

- GitHub 連携で push 時に自動デプロイ
- 環境変数（Clerk, Supabase）を Vercel に設定

- セキュリティ：学校ID検証、画像アップロード検証、コンテンツ長制限
